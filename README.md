# Organic Flow Foundation

This is the first repository following the Organic Flow Knowledge Branching methodology. It contains the foundational knowledge for the Organic Flow Development approach.

## Repository Structure

This repository follows a unique branching strategy where:
- **main branch** contains only knowledge (documentation)
- **experiment/** branches contain implementation attempts
- **release/** branches contain successful implementations
- **harvest/** branches extract knowledge from experiments

## Knowledge Organization

```
main/
├── seeds/          # Problems and opportunities we've discovered
├── roots/          # Assumptions we're testing
├── trunks/         # Validated requirements
├── knowledge/      # Harvested insights from experiments
└── implementations.yaml  # Registry of all implementation attempts
```

## What This Repository Contains

This repository documents the Organic Flow methodology itself, including:
- The philosophy of knowledge-first development
- The branching strategy for managing knowledge vs code
- Patterns and practices for software development
- Case studies of applying the methodology

## Key Principle

**Knowledge is eternal, code is temporary.** This repository prioritizes capturing understanding over implementation details.

## Getting Started

1. Read the Seeds to understand what problems Organic Flow solves
2. Review the Roots to see our assumptions
3. Study the Trunks for validated approaches
4. Check implementations.yaml to see how we've tested our ideas

## Note on Experiments

Implementation code lives in experiment branches, not in main. To see how we've implemented tools or tested ideas, check out the relevant experiment branches listed in implementations.yaml.
