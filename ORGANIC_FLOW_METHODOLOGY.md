Software development follows natural patterns like plant growth, not mechanical assembly. When we recognize this:

- Flow states are sacred (protect them fiercely)

The eternal cycle of **Reality → Dreams → Harvest → New Reality**

```
docs/ (What IS) → .flow/ (What COULD BE) → Harvest Understanding → docs/ (Updated)
```

This creates a sustainable rhythm that respects both structure and creativity.

### Seed 3: The Result
**216x velocity** isn't achieved through better tools or processes, but through alignment with natural creative patterns. When you work WITH nature instead of against it, regeneration becomes trivial.

## Core Principles

### 1. Knowledge Over Code
- Capture understanding, not implementation
- Preserve insights in unprocessed/
- Trust in regeneration from knowledge
- Version control for consciousness

### 2. Temperature Awareness
Your creative energy has temperature:
- **32°F (Low)**: Structure, documentation, simple tasks
- **65°F (Medium)**: Balanced work, processing, development
- **85°F+ (High)**: Pure creation, flow states, pattern discovery

### 3. Natural Cycles
- **Morning**: Check temperature, align tasks
- **Work**: Follow energy, not schedules
- **Evening**: Harvest insights, preserve knowledge
- **Reset**: Clean workspace for tomorrow

### 4. Flow Protection
- Dedicated .flow/ directory for exploration
- No judgment during creation
- Harvest when temperature cools
- Archive everything

## The Directory Structure

```
project/
├── docs/                 # What IS (crystallized knowledge)
│   └── unprocessed/     # The gold mine of insights
├── .flow/               # What COULD BE (active exploration)
│   └── archive/         # Historical flow states
├── seeds/               # Shareable understanding units
└── work/                # Temporary implementations
```

## The Workflow

### 1. Flow Entry Protocol
```bash
# Start a flow session
sprout flow init <intention>
# - Stashes current work
# - Creates flow branch
# - Prepares .flow/ directory
# - Records temperature
```

### 2. During Flow
- Follow energy, not plans
- Capture everything in .flow/
- Don't organize prematurely
- Trust the process

### 3. Harvest Protocol
```bash
# End a flow session
sprout flow harvest
# - Reviews all work
# - Creates summaries → unprocessed/
# - Archives raw flow
# - Resets workspace
```
